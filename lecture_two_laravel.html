Form Requests & Custom Validation Rules

Implement robust, reusable validation workflows in Laravel using Form Requests, Rule objects, and custom rule classes.

Difficulty: Intermediate Duration: 45 min Validation

Introduction

Learn how to centralize validation, keep controllers clean, and create readable, testable validation rules with Laravel Form Requests and custom rules.

Main Concepts

Generate a dedicated request class to hold your validation and authorization logic.

php artisan make:request StorePostRequest
// app/Http/Requests/StorePostRequest.php
namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rule;

class StorePostRequest extends FormRequest
{
    public function authorize(): bool
    {
        // Only allow users who can create posts
        return $this->user()?->can('create', \App\Models\Post::class) ?? false;
    }

    public function rules(): array
    {
        return [
            'title' => ['required', 'string', 'min:5', 'max:120', Rule::unique('posts', 'title')],
            'content' => ['required', 'string', 'min:20'],
            'status' => ['required', Rule::in(['draft','published'])],
            'tags' => ['nullable', 'array'],
            'tags.*' => ['string', 'max:20'],
        ];
    }
}
  1. authorize() centralizes permission checks per request.
  2. rules() defines constraints with rule strings and Rule objects.
  3. Nested rules like tags.* validate each item.
يعني إيه Form Request؟

اعتبرها استمارة ذكية. بدل ما تكتب الفاليديشن في الكنترولر، بتحطه في كلاس مخصوص يمسك القواعد والتصاريح في مكان واحد فالكود يبقى منظم ونضيف.

// app/Http/Controllers/PostController.php
use App\Http\Requests\StorePostRequest;
use App\Models\Post;

public function store(StorePostRequest $request)
{
    $validated = $request->validated();
    $post = Post::create($validated);
    return redirect()->route('posts.show', $post);
}
  1. Type-hint the request to trigger validation automatically.
  2. Use $request->validated() to get safe data only.

php artisan make:rule Uppercase
// app/Rules/Uppercase.php
namespace App\Rules;

use Closure;
use Illuminate\Contracts\Validation\ValidationRule;

class Uppercase implements ValidationRule
{
    public function validate(string $attribute, mixed $value, Closure $fail): void
    {
        if ($value !== mb_strtoupper($value, 'UTF-8')) {
            $fail('The :attribute must be uppercase.');
        }
    }
}
// In rules()
return [
    'title' => ['required', new \App\Rules\Uppercase],
];
يعني إيه Validation Rule؟

قانون صغير بيشيّك على قيمة معينة. زي موظف الاستقبال اللي بيتأكد إن بياناتك كاملة ومظبوطة قبل ما تدخل.

use Illuminate\Validation\Rule;

// Updating a post: ignore the current model id
$rules = [
    'title' => [
        'required', 'string', 'min:5',
        Rule::unique('posts', 'title')->ignore($post->id)
    ],
];
// Conditional validation with sometimes()
use Illuminate\Support\Facades\Validator;

$validator = Validator::make($data, [
    'has_image' => ['boolean'],
    'image' => ['nullable', 'image']
]);

$validator->sometimes('image', 'required', function ($input) {
    return $input->has_image === true;
});
  1. Use Rule::unique()->ignore() to handle edits without false positives.
  2. Validator::sometimes adds rules only when conditions are met.

Real-life Analogies

Form Request

Like a checklist at reception; you can’t proceed unless all fields are correct.

authorize()

A security gate that checks if you’re allowed to submit this form at all.

Advanced Developer Tips

  • Translate error messages with lang/en/validation.php and custom keys.
  • Prefer rule objects for complex constraints and reusability.
  • Keep authorize() fast; move heavy logic into policies.
  • Use bail to stop on first failure for performance-critical fields.

Mini Project: Tasks Manager

Build a small tasks manager with fields: title, description, status, due_date, and optional tags.

  • Form Request handles all validation and authorization.
  • Custom rule validates that due_date is a business day.
  • Update scenario uses Rule::unique()->ignore($task->id).
php artisan make:request StoreTaskRequest
php artisan make:rule BusinessDay
